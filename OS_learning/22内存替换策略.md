# 第22章 超越物理内存：策略

## 一、本章核心问题

当物理内存充足时，发生缺页异常后，操作系统只需要从空闲页列表中找一个空闲页，把需要的页调入内存即可。

但是，当物理内存不足时，操作系统必须先从内存中踢出某些页，为新的页腾出空间。

这个过程涉及几个重要概念：

| 概念                        | 含义                             |
| --------------------------- | -------------------------------- |
| 换出 paging out             | 把内存中的页写回磁盘             |
| 踢出 evict                  | 从物理内存中移除某个页           |
| 替换策略 replacement policy | 决定踢出哪个页的策略             |
| 内存压力 memory pressure    | 物理内存不足导致系统需要频繁换页 |

本章的关键问题是：

> 操作系统应该踢出哪一页？

------

## 二、物理内存可以看成缓存

虚拟内存中的所有页不可能全部同时放入物理内存。

因此，可以把物理内存理解为虚拟页的缓存：

- 页在内存中：缓存命中
- 页不在内存中：缓存未命中，也就是缺页

替换策略的目标是：

> 尽量提高命中率，减少缺页次数。

因为缺页往往需要访问磁盘，而磁盘访问比内存访问慢得多。

------

## 三、AMAT：平均内存访问时间

AMAT，全称是 Average Memory Access Time，即平均内存访问时间。

公式为：

```text
AMAT = PHit × TM + PMiss × TD
```

其中：

| 符号  | 含义           |
| ----- | -------------- |
| PHit  | 命中概率       |
| PMiss | 未命中概率     |
| TM    | 访问内存的时间 |
| TD    | 访问磁盘的时间 |

并且：

```text
PHit + PMiss = 1
```

假设：

```text
TM = 100ns
TD = 10ms
PHit = 0.9
PMiss = 0.1
```

那么：

```text
AMAT = 0.9 × 100ns + 0.1 × 10ms
     ≈ 1ms
```

虽然未命中率只有 10%，但由于磁盘访问太慢，平均访问时间会被明显拉高。

所以，在虚拟内存系统中，提高命中率非常重要。

------

## 四、最优替换策略 OPT / MIN

最优替换策略由 Belady 提出，也叫 MIN 策略。

它的核心思想是：

> 踢出未来最晚才会被访问的页。

也就是说，如果必须牺牲一个页，就选择那个最长时间以后才会再次使用的页。

优点：

- 理论上缺页次数最少
- 可以作为其他算法的比较标准

缺点：

- 需要知道未来的访问序列
- 现实操作系统无法实现

因此，OPT 的意义主要是作为理论参考。

------

## 五、缓存未命中的三种类型

缓存未命中可以分为三类，也叫 3C：

| 类型         | 英文            | 含义                           |
| ------------ | --------------- | ------------------------------ |
| 强制性未命中 | Compulsory Miss | 第一次访问某页，缓存中原本没有 |
| 容量未命中   | Capacity Miss   | 缓存太小，无法容纳所有需要的页 |
| 冲突未命中   | Conflict Miss   | 硬件缓存位置限制导致冲突       |

在操作系统页缓存中，一般没有冲突未命中。

因为页面可以放入任意物理页框中，属于全相联结构。

------

## 六、FIFO：先进先出替换策略

FIFO，全称 First-In First-Out。

核心思想是：

> 谁最早进入内存，就先踢出谁。

优点：

- 实现简单
- 维护成本低

缺点：

- 不考虑页面是否重要
- 可能踢出马上还要使用的页
- 可能出现 Belady 异常

例如，某个页虽然最近频繁被访问，但如果它最早进入内存，FIFO 仍然可能把它踢出。

------

## 七、Belady 异常

一般情况下，缓存越大，命中率应该越高。

但是 FIFO 可能出现一种反常现象：

> 缓存变大后，缺页次数反而增加。

这叫 Belady 异常。

经典访问序列：

```text
1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5
```

FIFO 在缓存大小从 3 增加到 4 时，可能表现反而变差。

原因是 FIFO 没有栈特性。

栈特性指的是：

> 大小为 N+1 的缓存，一定包含大小为 N 的缓存内容。

LRU 具有栈特性，因此不会出现 Belady 异常。

------

## 八、Random：随机替换策略

Random 策略的核心思想是：

> 内存满时，随机选择一个页踢出。

优点：

- 实现非常简单
- 不容易被某些固定访问模式完全克制

缺点：

- 表现不稳定
- 不利用历史信息
- 可能随机踢出重要页面

Random 有时比 FIFO 好，有时比 FIFO 差，完全取决于运气。

------

## 九、LRU：最近最少使用策略

LRU，全称 Least-Recently-Used。

核心思想是：

> 踢出最长时间没有被访问过的页。

LRU 利用了程序的时间局部性：

> 最近访问过的页，很可能不久之后还会再次访问。

优点：

- 通常比 FIFO 和 Random 表现更好
- 利用了程序访问历史
- 不会出现 Belady 异常

缺点：

- 精确实现成本很高
- 对循环顺序访问不友好

LRU 不一定总是最优，但在很多实际程序中效果较好。

------

## 十、LFU：最不经常使用策略

LFU，全称 Least-Frequently-Used。

核心思想是：

> 踢出访问次数最少的页。

它利用的是频率信息。

优点：

- 能保留长期频繁访问的页

缺点：

- 容易被旧历史误导
- 曾经很热但现在不用的页，可能仍然留在内存中

因此，LFU 不一定比 LRU 更好。

------

## 十一、局部性原则

LRU 和 LFU 的基础是局部性原则。

局部性原则认为：

> 程序倾向于反复访问某些代码和数据。

局部性分为两种：

| 类型       | 含义                                   | 例子               |
| ---------- | -------------------------------------- | ------------------ |
| 时间局部性 | 最近访问过的内容可能很快再次访问       | 循环变量、函数代码 |
| 空间局部性 | 访问某个地址后，附近地址可能也会被访问 | 顺序遍历数组       |

例如：

```c
for (int i = 0; i < n; i++) {
    sum += a[i];
}
```

这里既有时间局部性，也有空间局部性。

------

## 十二、不同工作负载下的表现

### 1. 无局部性工作负载

无局部性指的是：

> 每次访问的页都是随机的，过去访问历史对未来没有参考价值。

特点：

- LRU 优势不明显
- FIFO、Random、LRU 表现差不多
- 命中率主要由缓存大小决定

结论：

> 如果程序没有局部性，聪明算法和简单算法区别不大。

------

### 2. 80—20 工作负载

80—20 工作负载指的是：

```text
80% 的访问集中在 20% 的页上
20% 的访问分散在剩下 80% 的页上
```

这是一种有局部性的访问模式。

在这种情况下：

- LRU 通常比 FIFO 更好
- LRU 通常比 Random 更好
- OPT 仍然最好

原因是：

> LRU 更容易保留经常访问的热门页。

------

### 3. 循环顺序工作负载

循环顺序工作负载指的是程序按顺序访问一组页，然后不断重复。

例如：

```text
0, 1, 2, 3, ..., 49
0, 1, 2, 3, ..., 49
...
```

如果缓存大小是 49，而循环访问 50 个页，LRU 可能表现非常差。

原因是：

> LRU 总是踢出最久没使用的页，但在循环访问中，最久没使用的页恰好马上就要被访问。

所以循环顺序访问是 LRU 和 FIFO 的弱点。

有趣的是，在这种场景下，Random 有时反而比 LRU 和 FIFO 更好。

------

## 十三、扫描抗性

扫描抗性，英文是 scan resistance。

它指的是：

> 页面替换算法面对顺序扫描、大循环访问时，不应该轻易把真正重要的页全部冲掉。

普通 LRU 容易被顺序扫描污染缓存。

例如，程序扫描一个很大的数组或文件时，LRU 可能会不断把原本重要的热页踢出去。

现代算法如 ARC 会尝试增强扫描抗性。

------

## 十四、精确 LRU 为什么难实现

精确 LRU 要求系统在每次内存访问时，都更新页面的最近使用状态。

但内存访问非常频繁，包括：

- 取指令
- 读数据
- 写数据
- 栈访问
- 堆访问

如果每次访问都要维护 LRU 链表或时间戳，开销会非常高。

一种可能实现方式是：

> 硬件在每次访问页时记录时间戳。

但是当需要替换页面时，操作系统仍然可能需要扫描大量页，找出时间最老的那个。

例如：

```text
物理内存：4GB
页大小：4KB
页数量：4GB / 4KB = 1,048,576 页
```

每次替换都扫描上百万页，成本太高。

因此现实系统通常不实现精确 LRU，而是使用近似 LRU。

------

## 十五、近似 LRU

近似 LRU 的目标是：

> 不一定找到绝对最久没用的页，只要找到一个差不多很久没用的页即可。

实现近似 LRU 通常需要硬件提供一个使用位。

使用位也叫引用位：

```text
use bit / reference bit
```

规则：

| 事件           | 操作               |
| -------------- | ------------------ |
| 页被读         | 硬件将使用位置为 1 |
| 页被写         | 硬件将使用位置为 1 |
| 操作系统检查后 | 可以将使用位清零   |

硬件负责置 1，操作系统负责清 0。

------

## 十六、时钟算法 Clock Algorithm

时钟算法是经典的近似 LRU 实现。

它的核心思想是：

> 把所有页组织成一个循环列表，用一个指针像时钟指针一样扫描。

这个指针叫 clock hand。

当需要替换页面时：

1. 检查当前页的使用位。
2. 如果使用位是 1，说明最近访问过，不踢出，把使用位清 0，指针后移。
3. 如果使用位是 0，说明最近没有访问过，可以踢出。

伪代码：

```c
while (true) {
    if (page[hand].use == 0) {
        victim = page[hand];
        break;
    } else {
        page[hand].use = 0;
        hand = (hand + 1) % num_pages;
    }
}
```

时钟算法也可以理解为第二次机会算法：

> 最近访问过的页会被放过一次，如果之后还没被访问，下一次就可能被踢出。

------

## 十七、考虑脏页

页面替换时，还要考虑页是否被修改过。

| 类型              | 含义       | 踢出成本     |
| ----------------- | ---------- | ------------ |
| 干净页 clean page | 没有被修改 | 可以直接丢弃 |
| 脏页 dirty page   | 被修改过   | 必须写回磁盘 |

脏页被踢出前必须写回磁盘，因此成本更高。

为了支持这个判断，硬件通常提供修改位，也叫脏位：

```text
modified bit / dirty bit
```

当页面被写入时，硬件会把脏位置为 1。

改进版时钟算法会优先选择：

```text
未使用 + 干净
```

也就是：

```text
use = 0 且 dirty = 0
```

这种页面既最近没用过，又不需要写回磁盘，最适合踢出。

------

## 十八、其他虚拟内存策略

页面替换不是虚拟内存系统的唯一策略。

操作系统还要考虑：

1. 什么时候把页载入内存
2. 是否提前加载某些页
3. 什么时候把页写回磁盘
4. 一次写回多少页

------

### 1. 按需分页 Demand Paging

按需分页指的是：

> 只有当页真正被访问时，才把它载入内存。

优点：

- 节省内存
- 避免加载无用页面
- 程序启动可以更快

缺点：

- 第一次访问某页时会发生缺页

------

### 2. 预取 Prefetching

预取指的是：

> 操作系统提前预测某些页即将被访问，并提前加载它们。

例如：

如果代码页 P 被访问，系统可能猜测 P+1 也很快会被访问，于是提前加载 P+1。

预取的关键是预测准确。

如果预测失败，会浪费：

- 内存空间
- 磁盘 I/O
- CPU 时间

所以预取不是越多越好。

------

### 3. 聚集写入 Clustering

页面写回磁盘时，可以一个一个写，也可以多个一起写。

多个页一起写叫：

```text
clustering / grouping
```

也就是聚集写入或分组写入。

原因是：

> 对磁盘来说，一次大的连续写入通常比多次小写入更高效。

尤其是传统机械硬盘，小随机 I/O 代价很高。

------

## 十九、抖动 Thrashing

抖动指的是：

> 系统内存严重不足，进程不断发生缺页、换入、换出，几乎无法执行有效工作。

例如：

```text
进程 A 需要 500MB 活跃内存
进程 B 需要 500MB 活跃内存
进程 C 需要 500MB 活跃内存

但物理内存只剩 800MB
```

这时所有进程的活跃页面集合无法同时放入内存。

结果是：

- A 刚换入页面
- B 又把 A 的页面挤出去
- C 又把 B 的页面挤出去
- A 再次运行时又缺页

系统大部分时间都花在换页上，而不是执行程序。

这就是抖动。

------

## 二十、工作集 Working Set

工作集指的是：

> 一个进程当前活跃使用的页面集合。

如果所有进程的工作集总和超过可用物理内存，就容易发生抖动。

可以粗略理解为：

```text
所有进程工作集大小 > 可用物理内存
```

此时系统会频繁换页，性能急剧下降。

------

## 二十一、准入控制 Admission Control

应对抖动的一种方式是准入控制。

准入控制的思想是：

> 与其让所有进程一起运行但都跑不动，不如减少同时运行的进程数量。

操作系统可以暂停或换出部分进程，让剩下进程的工作集能够放入内存，从而真正取得进展。

这体现了一个重要思想：

> 少做一点工作，有时比同时做太多工作更高效。

------

## 二十二、OOM Killer

一些现代系统在内存严重不足时，会采用更强硬的方式。

例如 Linux 中的：

```text
OOM Killer
```

全称：

```text
Out-Of-Memory Killer
```

它会在系统内存严重不足时，选择某个进程杀掉，以释放内存。

优点：

- 快速缓解内存压力

缺点：

- 可能杀掉重要进程
- 可能导致用户程序或系统组件异常退出

OOM Killer 可以理解为系统内存不足时的最后手段。

------

## 二十三、几种策略对比

| 策略   | 核心思想             | 优点                           | 缺点                           |
| ------ | -------------------- | ------------------------------ | ------------------------------ |
| OPT    | 踢出未来最晚访问的页 | 理论最优                       | 无法实现                       |
| FIFO   | 踢出最早进入内存的页 | 简单                           | 可能踢出重要页，有 Belady 异常 |
| Random | 随机踢出             | 简单，不容易被固定模式完全克制 | 表现不稳定                     |
| LFU    | 踢出访问次数最少的页 | 利用频率信息                   | 可能被旧历史误导               |
| LRU    | 踢出最近最少使用的页 | 通常效果好，不会 Belady 异常   | 精确实现成本高，怕循环扫描     |
| Clock  | 用使用位近似 LRU     | 成本低，现实可行               | 不如精确 LRU 准确              |

------

## 二十四、本章重点总结

本章讨论的是：

> 当物理内存不够时，操作系统如何选择被踢出的页。

核心思想是：

```text
物理内存 = 虚拟页的缓存
页面替换策略 = 缓存淘汰策略
目标 = 减少缺页，提高命中率
```

需要掌握的重点：

1. 缺页代价很高，因为可能需要访问磁盘。
2. OPT 理论最优，但无法实现。
3. FIFO 简单，但可能出现 Belady 异常。
4. Random 简单，但表现靠运气。
5. LRU 利用时间局部性，通常效果较好。
6. LRU 不会出现 Belady 异常，但怕循环顺序访问。
7. 精确 LRU 实现成本太高。
8. 现实系统常用 Clock 等算法近似 LRU。
9. 使用位用于判断页面最近是否被访问。
10. 脏位用于判断页面是否需要写回磁盘。
11. 操作系统更愿意踢出干净页。
12. 按需分页是常见的页面载入策略。
13. 预取可以提高性能，但预测失败会浪费资源。
14. 聚集写入可以提高磁盘写入效率。
15. 抖动表示系统大部分时间都在换页，而不是执行程序。
16. 内存严重不足时，系统可能使用准入控制或 OOM Killer。

------

## 二十五、考试常见问法

### 1. 什么是页面替换？

页面替换是指当物理内存不足时，操作系统选择某个页从内存中移除，为新页腾出空间。

------

### 2. 什么是最优替换策略？

最优替换策略会踢出未来最晚才会被访问的页。

它理论上缺页次数最少，但现实中无法实现，因为操作系统不知道未来。

------

### 3. FIFO 有什么问题？

FIFO 只根据进入内存的先后顺序替换页面，不考虑页面是否重要。

它可能踢出马上还要使用的页，并且可能出现 Belady 异常。

------

### 4. 什么是 Belady 异常？

Belady 异常指的是：

> 对某些替换算法来说，增加缓存大小后，缺页次数反而增加。

FIFO 可能出现 Belady 异常，LRU 不会。

------

### 5. LRU 为什么通常有效？

因为它利用了时间局部性。

最近访问过的页，很可能不久之后还会再次访问。

------

### 6. 精确 LRU 为什么难实现？

因为精确 LRU 需要在每次内存访问时更新页面历史信息，开销非常大。

------

### 7. 时钟算法如何工作？

时钟算法用使用位近似 LRU。

- use = 1：最近访问过，清零并跳过
- use = 0：最近没有访问过，可以踢出

------

### 8. 为什么优先踢出干净页？

因为干净页没有被修改，可以直接丢弃。

脏页被修改过，踢出前必须写回磁盘，成本更高。

------

### 9. 什么是抖动？

抖动是指系统内存严重不足，导致进程不断换入换出页面，大部分时间花在处理缺页上，而不是执行有效工作。

## 二十六、一句话理解本章

页面替换就是在物理内存不够时决定“谁该被踢出去”：OPT 看未来，LRU 看过去，Clock 用使用位低成本模仿 LRU，而现实系统还要考虑脏页、预取、写回和抖动。
