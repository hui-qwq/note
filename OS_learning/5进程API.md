# 第 5 章：插叙——进程 API 总结

## 1. 本章核心

本章介绍 UNIX/Linux 中最重要的一组进程控制 API：

- `fork()`：创建新进程。
- `wait()` / `waitpid()`：等待并回收子进程。
- `exec()` 系列函数：让当前进程执行另一个程序。
- `open()` / `close()` / `dup2()`：配合文件描述符实现重定向。
- `pipe()`：实现进程之间的数据传递。
- `kill()`：向进程发送信号。

本章最重要的思想是：

> UNIX 将“创建进程”和“运行新程序”分成了两步：`fork()` 和 `exec()`。这样 shell 就可以在 `fork()` 之后、`exec()` 之前修改子进程的环境，从而实现重定向、管道、后台运行等功能。

## 2. 最终记忆版

```text
fork()：复制当前进程，创建子进程。
wait()：父进程等待并回收子进程。
waitpid()：更精确地等待指定子进程。
exec()：把当前进程替换成另一个程序。
open()/close()/dup2()：控制文件描述符，实现重定向。
pipe()：连接两个进程，实现进程间通信。
kill()：向进程发送信号。
```

shell 的核心模型：

```text
shell 读取命令
fork 创建子进程
子进程在 exec 前修改文件描述符
子进程 exec 执行命令
父进程 wait 等待命令结束
shell 继续接收下一条命令
```

一句话总结：

> `fork()` 让 shell 保住自己，`exec()` 让子进程变成目标程序，`wait()` 让 shell 等命令结束，`dup2()` 和 `pipe()` 让重定向与管道成为可能。
