# 第15章 机制：地址转换

## 15.5 操作系统的问题与 15.6 小结

## 一、核心思想

在动态重定位中，硬件提供了 **基址寄存器 base** 和 **界限寄存器 bound**，可以快速完成地址转换和越界检查。

但是，光有硬件还不够。操作系统必须在关键时刻介入，负责内存管理、进程切换、异常处理等工作。

可以概括为：

```text
硬件负责：快速地址转换和越界检查
操作系统负责：分配、回收、保存、恢复、处理异常
```

动态重定位仍然遵循 **受限直接执行 LDE** 的思想：

```text
大部分时间让进程直接运行
关键时刻由操作系统介入控制
```

------

## 二、动态重定位中操作系统的职责

### 1. 进程创建时：分配物理内存

当一个新进程被创建时，操作系统需要为它的地址空间寻找一块空闲的物理内存。

在本章的简单假设中：

- 每个进程地址空间大小相同；
- 每个进程地址空间小于物理内存；
- 每个进程必须连续放在物理内存中。

因此，操作系统可以把物理内存划分成若干固定大小的槽块。

例如：

```text
0KB  ~ 16KB   操作系统
16KB ~ 32KB   空闲
32KB ~ 48KB   进程A
48KB ~ 64KB   空闲
```

这时空闲列表为：

```text
16KB ~ 32KB
48KB ~ 64KB
```

操作系统通过 **空闲列表 free list** 记录哪些物理内存还没有被使用。

------

### 2. 进程终止时：回收内存

当进程正常退出，或者因为越界访问等错误被强制终止时，操作系统要回收它占用的内存。

操作系统需要做：

```text
1. 回收该进程占用的物理内存
2. 将这块内存放回空闲列表
3. 清除进程相关的数据结构
4. 从进程表中移除该进程
```

也就是说，进程结束后，它原来占用的内存应该重新变成可分配状态。

------

### 3. 上下文切换时：保存和恢复 base/bound

每个 CPU 只有一对：

```text
base 寄存器
bound 寄存器
```

但是每个进程的基址和界限可能不同。

例如：

```text
进程A:
base = 32KB
bound = 16KB

进程B:
base = 48KB
bound = 16KB
```

当 CPU 从进程 A 切换到进程 B 时，操作系统必须：

```text
1. 保存进程 A 的 base/bound 到 A 的 PCB 中
2. 从进程 B 的 PCB 中恢复 B 的 base/bound
3. 再让进程 B 继续运行
```

如果切换进程时没有正确更新 base/bound，就会导致进程访问到错误的物理内存。

例如，进程 B 运行时如果还使用进程 A 的 base，那么进程 B 的虚拟地址就会被错误地翻译到进程 A 的物理内存中。

这会破坏内存保护。

------

### 4. 异常发生时：执行异常处理程序

如果用户程序访问非法地址，例如：

```text
bound = 16KB
虚拟地址 = 20KB
```

硬件发现地址越界后，会触发异常。

此时 CPU 会：

```text
1. 从用户模式切换到内核模式
2. 跳转到操作系统的异常处理程序
3. 由操作系统决定如何处理
```

通常情况下，操作系统会终止这个非法访问内存的进程。

类似地，如果用户程序试图执行特权指令，例如修改 base/bound 寄存器，也会触发异常。

------

## 三、动态重定位下的硬件和操作系统交互

### 1. 系统启动时

操作系统在启动时会完成一些初始化工作：

```text
1. 初始化陷阱表
2. 设置系统调用处理程序
3. 设置时钟中断处理程序
4. 设置非法内存访问处理程序
5. 设置非法指令处理程序
6. 初始化进程表
7. 初始化空闲列表
8. 启动时钟中断
```

这些工作为之后的进程运行和异常处理做好准备。

------

### 2. 启动进程 A

操作系统启动进程 A 时，需要：

```text
1. 在进程表中为 A 分配条目
2. 为 A 分配物理内存
3. 设置 A 的 base/bound 寄存器
4. 恢复 A 的寄存器状态
5. 从内核模式返回到用户模式
6. 跳转到 A 的程序计数器位置
```

之后进程 A 开始运行。

------

### 3. 进程 A 正常运行

进程 A 运行时，地址转换由硬件完成。

例如：

```text
虚拟地址 + base = 物理地址
```

如果进程访问的地址没有越界，硬件直接完成转换，不需要操作系统介入。

所以正常情况下：

```text
取指令：硬件完成地址转换
读数据：硬件完成地址转换
写数据：硬件完成地址转换
```

这就是动态重定位高效的原因。

------

### 4. 时钟中断发生，切换到进程 B

当时钟中断发生时，CPU 会进入内核模式，操作系统开始进行上下文切换。

操作系统会：

```text
1. 保存进程 A 的寄存器状态
2. 保存进程 A 的 base/bound
3. 从进程 B 的 PCB 中恢复寄存器状态
4. 恢复进程 B 的 base/bound
5. 从陷阱返回，进入用户模式
6. 继续运行进程 B
```

------

### 5. 进程 B 越界访问

如果进程 B 执行了一次错误的加载，例如访问超过 bound 的虚拟地址：

```text
虚拟地址 >= bound
```

硬件会触发异常。

然后操作系统会：

```text
1. 进入异常处理程序
2. 判断进程 B 发生非法内存访问
3. 终止进程 B
4. 回收进程 B 占用的内存
5. 从进程表中移除进程 B
```

------

## 四、动态重定位的优点

### 1. 高效

动态重定位只需要简单的硬件支持：

```text
物理地址 = 虚拟地址 + base
```

同时进行界限检查：

```text
0 <= 虚拟地址 < bound
```

这个过程非常快。

------

### 2. 透明

程序自己看到的地址空间仍然从 0 开始。

程序并不知道自己实际被放在物理内存的什么位置。

例如程序以为自己访问：

```text
虚拟地址 128
```

但实际硬件访问的是：

```text
物理地址 = 128 + base
```

这个过程对程序是透明的。

------

### 3. 提供保护

界限寄存器可以防止进程访问自己的地址空间之外的内存。

如果进程访问非法地址，硬件会触发异常，操作系统可以终止该进程。

这样可以防止：

```text
1. 进程访问其他进程的内存
2. 进程破坏操作系统内存
3. 进程随意修改关键系统数据结构
```

保护是操作系统非常重要的目标。

------

## 五、动态重定位的缺点：内部碎片

虽然基址加界限机制简单高效，但它也存在明显缺点：**内部碎片**。

在这种机制下，整个地址空间必须作为一整块连续区域放入物理内存。

例如一个进程的地址空间大小是 16KB：

```text
代码段：占用一小部分
堆：占用一小部分
中间空闲区域：大量未使用
栈：占用一小部分
```

但是操作系统仍然要为它分配完整的 16KB 物理内存。

于是，已经分配给进程的内存内部有大量空间没有真正使用。

这种浪费叫：

```text
内部碎片 internal fragmentation
```

内部碎片指的是：

```text
已经分配出去的内存块内部存在未使用空间
```

------

## 六、为什么接下来要讲分段？

基址加界限机制把整个地址空间看成一个整体：

```text
代码 + 堆 + 空闲区域 + 栈
```

然后一次性把它放入物理内存。

这样会浪费中间大量没有使用的空间。

为了减少浪费，可以把地址空间拆成多个逻辑部分：

```text
代码段
堆段
栈段
```

每个段单独进行重定位和界限检查。

这就是后面要学习的：

```text
分段 segmentation
```

分段可以看成是基址加界限机制的推广。

------

## 七、本节重点总结

动态重定位需要操作系统和硬件配合完成。

硬件负责：

```text
1. 保存 base/bound 寄存器
2. 每次内存访问时进行地址转换
3. 每次内存访问时进行越界检查
4. 发现非法访问时触发异常
```

操作系统负责：

```text
1. 创建进程时分配物理内存
2. 终止进程时回收物理内存
3. 维护空闲列表 free list
4. 上下文切换时保存和恢复 base/bound
5. 初始化陷阱表和异常处理程序
6. 处理非法内存访问异常
```

动态重定位的优点：

```text
1. 简单
2. 高效
3. 对程序透明
4. 提供基本的内存保护
```

动态重定位的缺点：

```text
1. 地址空间必须连续放入物理内存
2. 容易造成内部碎片
3. 内存利用率不高
```

## 八、考试常考点

### 1. OS 在动态重定位中的职责

重点记：

```text
创建进程：分配内存
终止进程：回收内存
上下文切换：保存/恢复 base 和 bound
异常发生：执行异常处理程序
```

------

### 2. 上下文切换为什么要保存 base/bound？

因为每个进程的地址空间被放在不同的物理内存位置。

如果切换进程时不更新 base/bound，新进程的虚拟地址可能会被翻译到旧进程的物理内存中，导致内存保护失效。

------

### 3. 什么是内部碎片？

内部碎片是指：

```text
已经分配出去的内存块内部，有一部分空间没有被使用
```

在基址加界限机制中，由于整个地址空间必须固定大小、连续分配，所以即使进程实际只使用一小部分空间，也要占用整个槽块，导致内部碎片。

------

### 4. 动态重定位为什么高效？

因为正常内存访问时，地址转换完全由硬件完成：

```text
物理地址 = 虚拟地址 + base
```

操作系统不需要参与每一次地址转换。

只有在系统调用、中断、异常等关键时刻，操作系统才介入。

------

### 5. 动态重定位和受限直接执行的关系

CPU 虚拟化中的受限直接执行思想是：

```text
大部分时间让程序直接运行
关键时刻操作系统介入
```

内存虚拟化中的动态重定位也类似：

```text
大部分内存访问由硬件直接转换
只有越界访问、上下文切换等情况由操作系统介入
```

------

## 九、一句话总结

动态重定位通过 **base 寄存器** 实现地址重定位，通过 **bound 寄存器** 实现访问保护；硬件负责快速转换，操作系统负责分配、回收、切换和异常处理。它简单高效，但由于整个地址空间必须连续分配，容易产生内部碎片，因此后面需要引入分段机制。
