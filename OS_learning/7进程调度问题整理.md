# 第 7 章 进程调度问题整理

## 基本公式

### 响应时间

```text
响应时间 = 第一次运行时间 - 到达时间
```

它衡量的是：任务到达系统之后，过了多久第一次被 CPU 执行。

### 周转时间

```text
周转时间 = 完成时间 - 到达时间
```

它衡量的是：任务从到达系统到最终完成，一共花了多久。

本文默认：

- 所有作业都在 `t = 0` 到达；
- 不考虑上下文切换开销；
- FIFO 顺序按照题目给出的顺序；
- 作业分别记为 A、B、C。

---

# 问题 1：使用 SJF 和 FIFO 调度长度都为 200 的 3 个作业

作业如下：

| 作业 | 运行长度 |
|---|---:|
| A | 200 |
| B | 200 |
| C | 200 |

因为三个作业长度完全相同，所以 SJF 和 FIFO 的调度结果相同。

调度顺序：

```text
A → B → C
```

| 作业 | 首次运行时间 | 响应时间 | 完成时间 | 周转时间 |
|---|---:|---:|---:|---:|
| A | 0 | 0 | 200 | 200 |
| B | 200 | 200 | 400 | 400 |
| C | 400 | 400 | 600 | 600 |

平均响应时间：

```text
(0 + 200 + 400) / 3 = 200
```

平均周转时间：

```text
(200 + 400 + 600) / 3 = 400
```

因此：

| 调度算法 | 平均响应时间 | 平均周转时间 |
|---|---:|---:|
| FIFO | 200 | 400 |
| SJF | 200 | 400 |

# 问题 2：作业长度分别为 100、200、300，使用 SJF 和 FIFO

作业如下：

| 作业 | 运行长度 |
|---|---:|
| A | 100 |
| B | 200 |
| C | 300 |

如果 FIFO 的到达顺序正好是：

```text
A → B → C
```

也就是从短任务到长任务，那么 FIFO 和 SJF 的调度顺序相同。

| 作业 | 首次运行时间 | 响应时间 | 完成时间 | 周转时间 |
|---|---:|---:|---:|---:|
| A | 0 | 0 | 100 | 100 |
| B | 100 | 100 | 300 | 300 |
| C | 300 | 300 | 600 | 600 |

平均响应时间：

```text
(0 + 100 + 300) / 3 = 133.33
```

平均周转时间：

```text
(100 + 300 + 600) / 3 = 333.33
```

因此：

| 调度算法 | 平均响应时间 | 平均周转时间 |
|---|---:|---:|
| FIFO | 133.33 | 333.33 |
| SJF | 133.33 | 333.33 |

注意：如果 FIFO 的顺序不是从短到长，例如长任务先到，那么 FIFO 的表现会比 SJF 差。

# 问题 3：使用 RR 调度程序，时间片为 1

## 情况一：三个作业长度都为 200

作业如下：

| 作业 | 运行长度 |
|---|---:|
| A | 200 |
| B | 200 |
| C | 200 |

RR 的时间片为 1，因此调度过程大致为：

```text
A B C A B C A B C ...
```

响应时间如下：

| 作业 | 首次运行时间 | 响应时间 |
|---|---:|---:|
| A | 0 | 0 |
| B | 1 | 1 |
| C | 2 | 2 |

平均响应时间：

```text
(0 + 1 + 2) / 3 = 1
```

完成时间如下：

| 作业 | 完成时间 | 周转时间 |
|---|---:|---:|
| A | 598 | 598 |
| B | 599 | 599 |
| C | 600 | 600 |

平均周转时间：

```text
(598 + 599 + 600) / 3 = 599
```

因此：

| 调度算法 | 平均响应时间 | 平均周转时间 |
|---|---:|---:|
| RR，时间片为 1 | 1 | 599 |

## 情况二：作业长度分别为 100、200、300

作业如下：

| 作业 | 运行长度 |
|---|---:|
| A | 100 |
| B | 200 |
| C | 300 |

RR 的时间片为 1。

响应时间如下：

| 作业 | 首次运行时间 | 响应时间 |
|---|---:|---:|
| A | 0 | 0 |
| B | 1 | 1 |
| C | 2 | 2 |

平均响应时间：

```text
(0 + 1 + 2) / 3 = 1
```

完成时间如下：

| 作业 | 完成时间 | 周转时间 |
|---|---:|---:|
| A | 298 | 298 |
| B | 499 | 499 |
| C | 600 | 600 |

平均周转时间：

```text
(298 + 499 + 600) / 3 = 465.67
```

因此：

| 调度算法 | 平均响应时间 | 平均周转时间 |
|---|---:|---:|
| RR，时间片为 1 | 1 | 465.67 |

这个结果说明：

```text
RR 响应时间好，但周转时间差。
SJF 周转时间好，但响应时间差。
```

# 问题 4：对于什么类型的工作负载，SJF 提供与 FIFO 相同的周转时间？

当 FIFO 的到达顺序本来就是从短任务到长任务时，SJF 和 FIFO 的调度顺序相同。

例如：

```text
100 → 200 → 300
```

FIFO 按照这个顺序执行，SJF 也按照这个顺序执行，因此两者周转时间相同。

还有一个特殊情况：

```text
所有作业长度都相同。
```

例如：

```text
200, 200, 200
```

这种情况下，不管怎么排序，周转时间都一样。

总结：

> 当作业按照运行时间从短到长到达，或者所有作业长度相同，SJF 和 FIFO 的周转时间相同。

---

# 问题 5：对于什么类型的工作负载和量子长度，SJF 与 RR 提供相同的响应时间？

核心条件是：

> RR 的时间片足够大，使得每个短作业可以在一个时间片内直接运行完。

如果时间片 `q` 大于等于所有作业长度，那么 RR 不会真正发生轮转，而是退化成 FIFO。

如果此时作业顺序又刚好是从短到长，那么：

```text
RR = FIFO = SJF
```

例如作业为：

| 作业 | 运行长度 |
|---|---:|
| A | 100 |
| B | 200 |
| C | 300 |

如果时间片满足：

```text
q >= 300
```

那么 RR 的执行顺序就是：

```text
A → B → C
```

这和 SJF 的执行顺序相同，因此响应时间也相同。

总结：

> 当 RR 时间片足够大，并且作业顺序与 SJF 的短作业优先顺序一致时，SJF 和 RR 的响应时间相同。

# 问题 6：随着工作长度增加，SJF 的响应时间会怎样？

随着工作长度增加，SJF 的响应时间也会增加。

原因是 SJF 是非抢占式调度。它会运行完一个作业，再运行下一个作业。后面的作业必须等待前面的作业完成，才能第一次运行。

如果有 `N` 个长度相同的作业，每个作业长度为 `L`，那么响应时间分别为：

```text
0, L, 2L, 3L, ..., (N - 1)L
```

平均响应时间为：

```text
L × (N - 1) / 2
```

以 3 个作业为例：

| 作业长度 L | 响应时间 | 平均响应时间 |
|---:|---|---:|
| 100 | 0, 100, 200 | 100 |
| 200 | 0, 200, 400 | 200 |
| 300 | 0, 300, 600 | 300 |

因此：

> 在作业数量固定时，SJF 的平均响应时间和作业长度成正比。

---

# 问题 7：随着量子长度增加，RR 的响应时间会怎样？

RR 的时间片越大，响应时间通常越差。

原因是每个作业第一次运行之前，要等待排在它前面的作业各运行一个时间片。

假设有 `N` 个作业，时间片为 `q`，并且所有作业都足够长，那么最后一个作业的响应时间最差。

最坏响应时间为：

```text
最坏响应时间 = (N - 1) × q
```

平均响应时间为：

```text
平均响应时间 = q × (N - 1) / 2
```

例如有 3 个作业：

| 时间片 q | 响应时间 | 平均响应时间 | 最坏响应时间 |
|---:|---|---:|---:|
| 1 | 0, 1, 2 | 1 | 2 |
| 10 | 0, 10, 20 | 10 | 20 |
| 100 | 0, 100, 200 | 100 | 200 |

因此：

```text
时间片越大，RR 越接近 FIFO，响应时间越差。
时间片越小，RR 响应时间越好，但上下文切换开销越大。
```

# 总结

| 问题 | 核心答案 |
|---|---|
| 1 | 三个作业长度都是 200，FIFO 和 SJF 一样，平均响应时间为 200，平均周转时间为 400 |
| 2 | 作业长度为 100、200、300，且 FIFO 顺序从短到长时，FIFO 和 SJF 一样，平均响应时间为 133.33，平均周转时间为 333.33 |
| 3 | RR 时间片为 1 时，平均响应时间很好，为 1；但平均周转时间较差 |
| 4 | 当 FIFO 的到达顺序已经是短任务优先顺序时，SJF 和 FIFO 的周转时间相同 |
| 5 | 当 RR 时间片足够大，使其退化为 FIFO，且作业顺序与 SJF 一致时，SJF 和 RR 响应时间相同 |
| 6 | 随着工作长度增加，SJF 的响应时间线性增加 |
| 7 | 随着 RR 时间片增加，响应时间线性增加；最坏响应时间为 `(N - 1) × q` |

# 记忆口诀

```text
FIFO：谁先来谁先跑，简单但可能护航。
SJF：短任务先跑，周转时间优秀。
STCF：剩余时间最短先跑，是抢占式 SJF。
RR：大家轮流跑，响应时间好，但周转时间可能差。
```
