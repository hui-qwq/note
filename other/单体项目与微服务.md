# 单体项目与微服务

## 1. 单体项目

单体项目就是：**所有功能都写在一个 Spring Boot 项目里**。

例如：

```
FoodOrderSystem
├── 用户模块
├── 菜品模块
├── 订单模块
└── 支付模块
```

特点：

- 只启动一个项目
- 只部署一个 jar 包
- 模块之间是 Java 方法调用
- 适合学习、课设、中小项目

优点：

- 简单
- 开发快
- 调试方便
- 部署容易

缺点：

- 项目大了容易臃肿
- 模块之间容易耦合
- 一个地方出问题可能影响整个系统

## 2. 微服务

微服务就是：**把一个大系统拆成多个小服务**。

例如：

```
user-service      用户服务
order-service     订单服务
dish-service      菜品服务
payment-service   支付服务
```

特点：

- 多个独立项目
- 每个服务可以单独启动、部署
- 服务之间通过 HTTP / RPC 调用
- 适合大型系统、多团队协作

优点：

- 职责清晰
- 可以单独扩容
- 可以独立开发和部署

缺点：

- 架构复杂
- 服务通信有成本
- 调试和运维更麻烦
- 需要学习网关、注册中心、配置中心等技术
