# Git 概述

Git是一种分布式版本控制系统, 用于高效管理项目

## 1.1 版本控制

版本控制就是管理文件不同版本的变化过程。

## 1.2 为啥要版本控制

因为有很多人开发， 你不能乱改， 而且也方便复原

## 1.3 版本控制工具

### 集中式版本控制工具
SVN， VSS ...
集中版本控制系统如 CVS、 SVN等， 会有一个单一的集中管理服务器， 保存所有文件的修订版本， 工作人员从客户端连接到服务器， 取出新版本或文件或者更新提交。

好处是方便管理， 坏处是单点故障问题，中央服务器挂了就完蛋了

### 分布式版本控制工具
Git 这种
版本控制在本地进行， 会有一个远程库， 服务器断网也是可以提交到本地来控制版本的， 每个客户端都是一个完整的项目

## 1.4 Git 工作机制

一般有三个阶段：
### 工作区
代码存放的磁盘目录位置

### xxxxxxxxxx git reset --hard 版本号bash
临时存储， 将工作区的代码添加到暂存区， 通过
```bash 
git add (位置)
```

### 本地库
提交到本地库的东西就生成版本了， 一般是删不掉的，通过
```bash
git commit
```
不过现在还是可以回退的， 通过
```bash
git reset --soft HEAD~1
```
其他删除方法听说也是有的， 当然也可以删库跑路。
