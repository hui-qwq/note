## 1.分支

可以开一个单独的副本

### 1.1 分支的好处
同时并行推进多个功能开发， 提高开发效率
一个分支开发失败不对其他分支有影响

### 1.2 分支的操作
查看分支
```bash
git branch -v
```

创建分支
```bash
git branch 分支名
```

切换分支
```bash
git checkout 分支名
```

分支合并
将指定分支合并到当前分支上
```bash
git merge 分支名 
```

合并冲突
当前分支和选择分支都修改， 并且修改的地方有重叠， 就会出现合并冲突
此时必选认为选择一个分支
届时会标记出冲突位置， 要人为调整代码，再次 add 并 commit
